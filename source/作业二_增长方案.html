好伴AI 增长方案

如果我有幸加入好伴担任产品经理，怎样打造产品促进业务增长
承接竞品与用户洞察，一条归因链：诊断问题 → 定战略 → 筛需求 → 转功能 → 获客变现 → 验证落地

1. 增长诊断

通过竞分已经诊断出好伴的处境：模型不弱，但陪伴链断裂、留不住人、变现从0、获客没入口。把这些放进增长漏斗看一遍，问题就清楚了——

增长环节好伴现状判断
获客没有支付宝/字节那样的流量入口弱，但不是最要问题（有营销资源）
激活（首次价值）解读能力强、循证、有温度✅ 好——模型与技术能力没问题
留存陪伴链断：刚传报告就忘、无跨时间主动关怀❌ 核心问题
变现29.9单次解读失败、从0开始❌ 留存没解决，变现无从谈起
裂变没有家庭关系链的裂变机制❌ 没启动

核心判断：好伴的增长不是"激活"出了问题（读报告足够强），是"留存"这一环断了。也就没有后续

所以增长的第一性问题很明确：先把留存（陪伴）修好，后面才谈得上变现和拉新。 这就是为什么三阶段路线必须"先止血、再增长、后变现"。

2. 增长战略

诊断指向"留存是问题、陪伴是解法"。战略就从这里推出来。

北极星指标

持续活跃的家庭健康档案数——有多少个家庭的健康档案在被持续维护、且在收到主动关怀并响应。

为什么是它：问题是留存、解法是陪伴，那衡量成败的就该是"陪伴有没有真正发生"。这个指标一个装下三件事——家庭（关系链）、档案（记忆/数据）、持续活跃（陪伴/留存）。指标上涨说明陪伴真的留住了用户。

增长公式

把北极星拆开，每一项都对应一个能下手的地方：

持续活跃家庭档案数 = 新增家庭数 × 激活率 × 持续活跃留存率 × (1 + 家庭裂变系数)

变现（结果）= 活跃家庭档案数 × 付费转化率 × 客单价 × 续费率

三阶段路线

阶段目标核心杠杆攻公式里的哪一项看什么指标
一、止血救住留存强化陪伴链：数据自动归档 + 跨会话真记忆↑ 持续活跃留存率档案自动归档率、次周留存
二、增长差异化拉新主动关怀（让陪伴可感知）+ 家庭健康（子女带父母）↑ 新增家庭数、↑ 家庭裂变系数新增家庭数、北极星
三、变现可持续的钱家庭健康订阅 + B端第二曲线↑ 付费转化率、↑ 续费率付费家庭数、续费率

  • P1（增长·紧随其后）：家庭多人档案 + 子女远程建档 + 父母长辈端 + 家庭简报 + 中立显性化 + 确定性表达 → 把"家庭健康"和差异化立起来，撬动子女付费者。

4. V2.0 产品方案

产品定位：从"一个会读报告的AI"，升级为"记得你全家、会主动管的健康管家"。读报告只是入口，留住人的是它之后持续的陪伴。

4.1 P0 · 做好陪伴链（止血）

陪伴链是四个环扣起来的：自动归档 → 真记忆 → 主动关怀 → 闭环。

① 数据自动归档

  • 痛点：现在上传默认"未选择档案"，数据不入档，记忆从源头就断了。
  • 设计：任何健康信息自动结构化进对应成员的档案，零手动操作——OCR识别报告归属人（姓名/年龄）自动归档；对话里提到的症状、用药、家族史，自动抽取入档。
  • 创新点（解"数据获取摩擦"死结）：让数据从三个口子无感流入——拍照即归档（拍报告自动入档+提指标）、对话即建档（聊天中自然提到的健康信息被动沉淀）、穿戴设备直接录入/一句话补充。档案因此从"一次性快照"变成"活的时间线"——这是主动关怀唯一的燃料。

② 跨会话真记忆——陪伴的内核

  • 把 Wise MemOS 的技术底牌真正做进产品：每次对话默认带全档案上下文，开口就是"结合你3月的血脂看这次的变化"。
  • 可感知设计：让用户看见它记得——主动引用历史、档案时间线可视化。
  • 直接修掉实测那个"刚传报告就忘"。

③ 主动关怀引擎——差异化和续费的核心

  • 从"被动应答"变"主动触达"：基于档案和趋势，主动发起复查提醒、指标趋势预警、关怀问候。
  • 分寸设计（避免变骚扰）：只在有医学依据时触达（复查到期 / 趋势异常 / 用药提醒），且每次触达必须可行动；设"价值密度"门槛，宁可少推不可滥推——健康场景信任透支一次就归零。

④ 解读后闭环——接住"然后呢"

  • 解读完不止给建议，一键转成行动：设复查提醒、把异常指标加入追踪、生成可勾选的健康计划。
  • 形成"解读 → 行动 → 追踪 → 下次解读"的循环，把一次性解读变成持续管理。

四环合起来：自动归档（燃料）→ 真记忆（内核）→ 主动关怀（触达）→ 闭环（承接）。这就是"陪伴链"，也是好伴从"读报告工具"变成"陪伴管家"的地基。

4.2 P1 · 立起家庭健康（增长）

好伴的差异化引擎，也是解"双边断点"（子女付费、父母不用）的地方。核心思路一句话：把"使用"和"管理"拆到两端——父母只管用，子女管管理和付费。

① 子女远程建档（付费者入口）：子女在自己手机上给父母建档——拍报告、录病史、勾选要盯的指标。父母不用任何操作，档案就建起来了。解决"父母不会建档"。

② 父母极简长辈端（使用者入口）：父母端极简，大字、语音优先、一个按钮"问健康"，甚至做成微信轻入口不用下载。父母只需要"用"，不需要"管"。解决"父母嫌麻烦不愿用"。

③ 子女端家庭健康简报（让付费者持续感到值——续费关键）：定期把父母的健康状况、异常、趋势生成简报推给子女。远在外地的子女看得见父母健康、感到被照顾到——这是子女愿意持续付费的核心理由。

三件一起，"子女装 → 父母用 → 子女持续付费"的协作闭环就转起来了。

4.3 中立与确定性（撑起差异化）

  • 中立显性化：把"只替你说话"做成用户能看见的——解读附证据来源、明确标注"我们不卖药不卖体检套餐"、建议绝不夹带推销。把好伴本就有的"循证可见"优势从隐性变显性。
  • 确定性表达：红黄绿分级（回答"我有没有大事"）+ 就医决策（挂哪科、要不要去）+ 基于档案的趋势预测。专治用户最焦虑的"我要不要慌"。

5. 获客与变现

好伴的获客和变现，跑在同一条链上——家庭关系链。它解的是两个风险：没流量入口（获客）、29.9失败从0（变现）。

5.1 获客与裂变（解"没入口"）

三条结构性低成本管道，不烧钱买量：

  • 家庭关系链裂变：一个付费子女带来一个家庭（3-5人档案），获客成本被家庭规模摊薄——大厂个人账户做不出的护城河。裂变钩子：邀请家人共管爸妈健康、家庭简报转发给兄弟姐妹、父母端告诉老伴。
  • B端医院导流：300医院、191家医团队是别人没有的线下入口。患者做完检查→引导用好伴解读→沉淀成C端档案→转家庭健康。把B端赋能用成C端获客管道。
  • 中立口碑内容：靠"不卖药只说真话"的信任内容拉新（对标丁香辟谣人设、OpenEvidence医生口碑），省广告费，而非烧钱买开屏。

5.2 变现设计（解"29.9从0"）

先避两条死路：29.9失败教训——单次付费、为信息付费、低频场景，三条都死；平安/薄荷的会员套路（投诉占比30%+）也死。所以变现要反着设计。

主模式 · 家庭健康订阅

  • 卖的是"持续安心"，不是"一次信息"（对标Hims订阅制、薄荷"卖结果"）。
  • 以家庭为单位订阅：一个子女付费、管全家。客单更高，且抗流失——为父母付费意愿远高于为自己，家庭账户天然黏。
  • 价值锚点：主动关怀 + 家庭简报 + 不限次解读 + 全家档案管理。
  • 透明、随时退——刻意反"套路"，避开平安薄荷的投诉死法。

第二曲线 · B端MaaS

  • 把模型和解读能力做成API，卖给医院、健康机构、保险（已落地300医院）。
  • 现金流比C端稳，是蚂蚁攻不进的纵深；而且B端既是收入、又反哺C端获客。

定价逻辑（诚实）：具体价格需用户付费意愿测试（作业1已标注缺直接调研），方向明确——做价值锚定而非成本定价：家庭年费锚"比一次三甲专家号便宜、比一次体检套餐便宜"，绝不再用29.9这种单点裸价。

5.3 一条链，一鱼两吃

家庭关系链同时驱动两件事——裂变拉来一个家庭（获客），家庭订阅收一个家庭的钱（变现）。获客和变现共用一条链、互相喂，这是好伴模式里效率最高、也最难被大厂复制的地方。

6. 验证与节奏

6.1 先验证最危险的假设

假设最小代价怎么验证通过标准
子女装了，父母真的会用（双边断点）招募50个家庭，子女建档，看父母端7日活跃父母7日活跃率达标
主动关怀能拉动留存（陪伴价值）A/B：有主动关怀 vs 没有，比次月留存留存显著提升
用户愿为家庭健康持续付费（续费）小范围放家庭订阅，看转化+3月续费续费率达标
数据能持续进来（数据摩擦）看自动归档率、档案周更新率档案活跃率达标

先验证前两个——"父母会不会用"和"主动关怀能不能留住人"，这俩错了，整套战略不成立。（具体阈值需用真实数据校准，这里给的是验证项和方向。）

6.2 MVP：先跑通陪伴链能不能留住人

不要一上来做全。最小闭环只验证一件事——陪伴链能不能留住人：

  • 做：自动归档 + 真记忆 + 基础主动关怀 + 一个家庭场景（子女给一位父母建档、收简报）。
  • 砍：复杂订阅、B端、多成员、长辈端完整功能，全部后置。
  • 目标：用最小功能跑通"建档 → 记忆 → 主动关怀 → 留存"，看北极星（活跃家庭档案）能不能涨起来。

6.3 里程碑节奏（对齐三阶段）

阶段周期做什么验证什么

6.4 风险应对

风险应对
父母不用（双边断点）MVP先验，不过关就再简化父母端 / 子女代操作更多
数据进不来（摩擦）多管道兜底（拍照+对话+可穿戴+B端导入），不押单一
续费低先用主动关怀+简报把价值感做实，再谈收费；订阅透明、随时退

收口：这套方案不是一次性赌全部，是"先验证最危险的假设 → 小闭环MVP → 分阶段放大"。每步都有验证关卡，错了能及时调，对了再加码。
